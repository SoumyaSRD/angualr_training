Dependency Injection (DI) in Angular

Dependency Injection (DI) is a core design pattern in Angular that allows classes to receive their required dependencies from external sources rather than creating them manually. This approach promotes loose coupling between different parts of an application, improves code reusability, and makes testing easier. Instead of instantiating services directly inside a component, Angular provides them automatically through its built-in dependency injection system.

In Angular, dependencies are typically services that contain shared logic, data access functionality, configuration, or state management. The Angular framework uses an injector to create and manage instances of these services and supply them wherever they are needed.

Why Dependency Injection is Important

  • Loose Coupling: Components do not need to know how to create their dependencies.
  • Reusability: Services can be reused across multiple components.
  • Maintainability: Easier to update or replace services without changing dependent components.
  • Testability: Dependencies can be mocked or replaced during testing.
  • Centralized Management: Angular manages lifecycle and instantiation automatically.

How Dependency Injection Works

Angular's DI system consists of three main parts:

  • Service: A class that provides functionality or data.
  • Injector: Responsible for creating and delivering service instances.
  • Consumer: A component, directive, pipe, or another service that requires the dependency.

When Angular creates a component, it checks the constructor parameters and automatically injects the required dependencies if they are registered with the injector.

Types of Providers in Angular

  • Root Provider: Registered globally using providedIn: 'root'.
  • Component Provider: Local instance scoped to component tree.
  • Module Provider: Used in legacy NgModule-based apps.

Injector Hierarchy

Angular uses a hierarchical injector system. Child components can access services from parent injectors or override them locally.

  • Root Injector — global application services.
  • Component Injectors — scoped services.

Modern Dependency Injection (Angular Standalone)

With standalone components, DI still works the same way. Angular also introduced the inject() function for functional injection. 

const dataService = inject(DataService);

Advanced DI Concepts

  • Injection Tokens: For non-class dependencies.
  • Optional Injection: Prevent errors if dependency missing.
  • Multi Providers: Multiple values under one token.
  • Factory Providers: Dynamic creation logic.

Best Practices

  • Use services for shared logic.
  • Prefer providedIn: 'root'.
  • Avoid heavy business logic in components.
  • Scope services only when needed.
  • Keep services single-responsibility.

Summary

Dependency Injection keeps Angular applications modular, scalable, and easy to maintain by separating dependency creation from usage.
