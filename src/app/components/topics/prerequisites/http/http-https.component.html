HTTP (HyperText Transfer Protocol) is the foundation of data communication on the web. It's an application-layer protocol that defines how messages are formatted and transmitted between web browsers and servers. HTTP follows a request-response model where clients send requests and servers return responses.

HTTPS (HTTP Secure) is the secure version of HTTP, using TLS/SSL encryption to protect data in transit. All modern web applications should use HTTPS to prevent man-in-the-middle attacks, protect user privacy, and meet security best practices. Browsers now mark HTTP sites as "Not Secure" to encourage HTTPS adoption.

HTTP Methods

  • GET: Retrieve data from server (idempotent, cacheable)
  • POST: Submit data to create new resources
  • PUT: Update existing resources (full replacement)
  • PATCH: Partial update of resources
  • DELETE: Remove resources from server
  • OPTIONS: Describe communication options (used in CORS)

Status Codes

HTTP responses include status codes indicating the result. 2xx codes indicate success (200 OK, 201 Created). 3xx codes are redirects (301 Moved Permanently). 4xx codes indicate client errors (404 Not Found, 400 Bad Request). 5xx codes indicate server errors (500 Internal Server Error, 503 Service Unavailable).
